① プロジェクトの背景・目的・目標

1-1. 背景：標準化された教育OSとの「構造的なミスマッチ」 現在の公教育システムは、産業社会に適した「均質な人材」の育成に最適化されています。この画一的な環境下では、ADHDやASDといった特異なパラメータを持つ子供たちは、本来は「異能（イノベーションの種）」であるにも関わらず、システムのエラーとして排除され、不登校や学習性無力感に追い込まれています。 文部科学省の調査によれば、2023年度の不登校児童生徒数は約30万人と過去最多を更新し続けており、通常学級における発達障害の推定割合は約8.8%（2022年調査）に上ります。これらの子どもたちに対する既存の支援は「矯正」を前提としたものが大半であり、その特性を「能力」として活用する仕組みは存在しません。 提案者（高崎）は、臨床心理士および高校教員として長年彼らと向き合い、500人以上の子どもの知能検査（WAIS/WISC）を実施・分析してきました。現場で確信しているのは、「彼らの能力が低いのではなく、既存の教育インターフェースが彼らの脳の仕様に適合していないだけである」という事実です。彼らに必要なのは、欠点を矯正する福祉ではなく、その突出した特性を最大限にドライブさせるための「専用環境の実装」です。 1-2. 目的：AIによる「精神と時の部屋」の社会実装 本プロジェクトの目的は、AIとメタバース技術を統合し、神経多様性児の才能を最短距離で覚醒させる適応型プラットフォーム『Genesis』を構築することです。コンセプトは、漫画『ドラゴンボール』に登場する「精神と時の部屋」の技術的な再現です。 ・環境の動的最適化：視線・生体ログ・操作履歴をマルチモーダルAIがリアルタイムに解析。ユーザーの認知特性に合わせて、情報の粒度や提示方法を動的に最適化します。 ・学習曲線のショートカット：苦手な科目の克服に時間を費やすのではなく、AIバディが「君の特性なら、このスキルセットだけ習得すれば勝てる」という最短ルートを提示し、圧倒的な時間短縮を実現します。 1-3. 目標：「保護対象」から「国家資源」への転換 最終ゴールは、教育から実社会（労働・経済）へシームレスに接続する「人材開発エコシステム」の確立です。 ・短期目標：ゲーム内での創作活動を通じて「実社会の価値（企業案件）」を生み出し、経済的な報酬を得られることを実証します。 ・中長期目標：「不登校＝弱者」という社会通念を覆し、彼らを「10代で起業し外貨を稼ぐトップクリエイター」として社会実装します。埋もれていた異能を「国家資源」へと転換する新たな社会OSを秋田から構築します。

② 開発に関する未踏性・期待される効果

本プロジェクトの最大の未踏性は、「検査」と「ゲーム」の境界を消失させる点にある。既存の知能検査（WAIS/WISC等）は被検者に強いストレスを与え、特に発達特性のある子どもは検査場面で本来の能力を発揮できないことが多い。本システムはメタバース内での自然な振る舞いから認知特性を逆算する「Stealth WAIS（ステルス・アセスメント）」技術を確立する。 2-1. 技術的コア①：Gamified Assessment Engine 提案者は臨床心理士として500人以上の子どもにWAIS/WISCを実施し、「検査数値」と「実際の行動特性」の対応関係を蓄積してきた。この臨床知見をアルゴリズムに変換するのが本システムの核心である。 具体的には、以下の3段階で認知特性を推定する： 【Step 1：行動指標の収集】 メタバース内でのゲームプレイ中に、以下のデータをリアルタイムに取得する。 ・マウス/コントローラーの移動軌跡・速度・加速度 ・クリック/タップの反応速度とタイミング精度 ・視線の移動パターン（注視点の滞留時間・サッケード特性） ・タスク切替時の遷移時間と誤操作率 ・指示理解から行動開始までの潜時 【Step 2：特徴量への変換】 収集した行動指標を、認知科学的に意味のある特徴量に変換する。 ・反応速度の分散 → 注意の持続性（不注意傾向の推定） ・マウス軌跡の滑らかさ → 視覚-運動協応の成熟度 ・聴覚指示 vs 視覚指示でのタスク達成率差 → 優位な入力モダリティ ・パターン記憶タスクの保持時間 → ワーキングメモリ容量 ・多重タスク時のエラー率変化 → 認知負荷への耐性 【Step 3：認知指標への逆推定】 特徴量の組み合わせから、WAIS/WISCの4指標に対応する認知プロファイルを推定する。 ・言語理解（VCI）：指示文の読解精度、語彙選択タスクの正答パターン ・知覚推理（PRI）：空間配置タスクの解法パターン、ブロック構成の効率 ・ワーキングメモリ（WMI）：NPCのパターン記憶タスクにおける系列再生精度 ・処理速度（PSI）：ターゲット捕捉の反応速度、視覚走査の効率性 この推定ロジックの根拠となるのは、提案者が500人以上の子どもに対して実施してきたWAIS/WISCの検査結果と、その子どもたちの実際のゲームプレイ・学習行動の観察記録である。 【クエストへの翻訳例】 ・処理速度（PSI）の測定 ⇒ クエスト「精霊の光集め」：ターゲット捕捉の反応速度とマウス軌跡から、視覚運動協応の特性を特定する。 ・ワーキングメモリ（WMI）の測定 ⇒ クエスト「魔法陣の模写」：NPCのパターン記憶タスクを通じ、聴覚的短期記憶の容量を測定する。 ・入出力特性（Modality）のABテスト ⇒ クエスト「ナビゲーション・チャレンジ」：「視覚のみ」と「聴覚のみ」でのクリアタイムを比較し、学習教材の提示形式（UI）を自動最適化する。 2-2. 技術的コア②：World as a Portfolio 履歴書では表現できない非認知能力を、3D空間上のオブジェクトとして可視化する。企業は「ワールドを探索する」ことで、その子のプログラミング能力や空間把握能力を直感的に評価できる。従来の文字ベースの履歴書・職務経歴書では伝えられない「空間認知能力」「創造性」「問題解決のプロセス」が、成果物そのものとして可視化される。 2-3. 技術的コア③：Synergy Install 個人の「欠落（凹）」を他者の「突出（凸）」で埋める、自律分散的な相互扶助システム。他者のスキルオブジェクトを自ワールドに「インストール」できる機能を実装し、「苦手なことは他者に頼る」という生存戦略をシステム的に学習させる。これは、発達特性のある子どもたちが社会で生きていく上で最も重要なスキルである「援助希求能力」の獲得を、ゲームメカニクスを通じて自然に促すものである。 2-4. 先行事例との差別化 【既存サービス（LITALICO、放課後デイ等）との比較】 ・アセスメント：検査室での標準化検査 → Genesis: ゲームプレイから自動推定（Stealth WAIS） ・個別最適化：支援者の経験に依存 → Genesis: AIが認知特性に基づき動的に最適化 ・成果の可視化：テストの点数 → Genesis: スキルオブジェクト（3D職務経歴書） ・社会接続：就労移行支援（限定的）→ Genesis: 企業タスクのクエスト化（BPO） ・スケーラビリティ：支援者1人あたり数名 → Genesis: AIにより人数制限なし

③ 開発の具体的な進め方

3-1. MVP（最小限の実証プロダクト）の定義 【MVPに含むもの】 ・Unityベースのメタバース空間（既存プロジェクト「Kitazuna」の基盤を活用） ・Stealth WAISの基本ロジック（処理速度・ワーキングメモリの2指標から開始） ・AIバディによるクエスト自動生成（LLM連携） ・スキルオブジェクトの自動生成と表示 ・if(塾)の生徒30名による実証テスト 【MVPに含まないもの（Phase 2以降）】 ・ブロックチェーンによるスキル証明（MVP段階ではサーバーサイドDBで管理） ・企業タスクとの自動マッチング ・グローバル展開向けの多言語対応 3-2. システム構成 ・Client Layer (Unity/C#): 没入型3D空間、操作ログ収集、リアルタイムUI最適化 ・Server Layer (Python/FastAPI): アセスメント推論エンジン、クエスト生成LLM連携 ・Data Layer (PostgreSQL/Firebase): ユーザーデータ、行動ログ、スキルオブジェクト管理 技術選定の根拠： ・Unity：チーム全員がUnity開発経験を持ち、既存プロジェクト「Kitazuna」（進捗50%）の資産を直接流用可能。 ・Python/FastAPI：AIモデルの推論パイプライン構築に最適。scikit-learn/PyTorchとの親和性が高い。 ・PostgreSQL：行動ログの時系列データと認知プロファイルの関係を柔軟にクエリするために必要。 3-3. 開発スケジュール（2026年7月〜2027年2月） 7月：キックオフ・基盤構築（Kitazunaからのコア資産移植完了、ログ収集基盤） 8月：Stealth WAIS α版（PSI・WMI測定クエスト3種の実装、if(塾)5名でのα版テスト） 9月：AIバディ統合（LLM連携によるクエスト自動生成、成功率80%以上） 10月：スキルオブジェクト実装（タスク達成→3Dオブジェクト自動生成） 11月：ギルド・シナジー機能（if(塾)30名全員での実証テスト） 12月：企業タスク接続（BPOクエスト化、1社以上の企業タスク稼働） 1月：最適化・改善（アセスメント精度の改善、UX改善） 2月：成果報告・事業化準備（実証データの分析、最初の売上達成） 3-4. 開発体制（Team Genesis） PM / Lead Developer：高崎 翔太 責務：アセスメントロジック設計、Unity開発、教師データ作成、AI実装 スキル：Unity/C#、ゲーム開発、AI活用開発、臨床心理（WAIS/WISC 500人以上実施） Content / AI Developer：山﨑 責務：カリキュラム設計、AIを活用したコンテンツ生成、サーバー運用 スキル：AI活用開発、Minecraft開発、サーバー運営 System / AI Developer：井上 責務：システム設計、データパイプライン構築、経済システム実装 スキル：AI活用開発、Minecraft開発、サーバー運営、数理設計 不足スキルへの対策： ・3Dモデリング：AIによる自動生成（Meshy等）を活用 ・データサイエンス：高崎が特徴量設計を担当、scikit-learn/AutoMLで対応 3-5. 技術的リスクと対策 ・アセスメント推定精度が不十分（高）→ if(塾)で実際のWISC検査結果との照合による精度検証 ・LLMによるクエスト生成の品質（中）→ プロンプトテンプレートによる品質管理 ・ユーザー数増加時のサーバー負荷（低）→ MVP段階は30名規模、クラウドで対応 3-6. 倫理的配慮・プライバシー対策 ・インフォームドコンセント：保護者および本人に書面で同意を取得 ・データの匿名化：認知プロファイルデータは個人を特定できない形で保存 ・データの最小化：認知特性推定に必要な最小限の行動ログに限定 ・アクセス制御：認知プロファイルへのアクセスは本人・保護者・担当支援者に限定 ・研究倫理審査：必要に応じて大学等のIRBへ申請

④ 提案者の実力・能力を示す実績

4-1. 臨床知見×技術開発の融合（高崎翔太） PM高崎は臨床心理士・高校教員として、以下の実績を持つ。 ・WAIS/WISC実施実績：500人以上の子どもに知能検査を実施・分析。検査数値と実際の行動特性の対応関係に関する膨大な臨床知見を蓄積しており、Stealth WAISのアセスメントロジック設計の根拠となる。 ・技術開発能力：Unityを用いたゲーム開発およびAIを活用した高度なシステム開発が可能。技術とドメイン知識の両方を持つことで、「臨床知見のアルゴリズム化」を一人で完結できる稀有な人材である。 ・教育現場でのDX実践：クラーク記念国際高等学校にて、ICTを活用した発達特性のある生徒への教育実践を日常的に行っている。 ・起業・ピッチ実績：EO North Japan主催「NEXT10」優勝、2024年度秋田若者チャレンジ採択、2025年度ASHIOTO採択 4-2. 検証フィールド「if(塾)」 チームメンバーが運営する学習塾「if(塾)」は、以下の特徴を持つ実証フィールドである。 ・生徒数：約30名 ・生徒の特性：全員が発達特性（ADHD/ASD等）または不登校経験を持つ ・検証可能性：プロトタイプを即座に生徒にテストし、高速にフィードバックループを回せる環境 ・教師データの蓄積：全生徒のWISC検査結果と日常の行動データが対応づけられており、Stealth WAISの精度検証に直接活用できる 4-3. チームメンバーの技術力 ・山﨑：AI活用による高度な開発能力を持ち、Minecraftの開発経験（プラグイン・Mod開発）およびサーバー運営の実績がある。大学では学生運営カフェの中核メンバーとして業務システムを自ら構築。 ・井上：AI甲子園（2025年度）全国5位入賞の実力者。将棋駒の画像認識AIを開発。Minecraftでのゲーム・Mod開発を自在にこなし、経済システムの数理モデル構築を担当する。 4-4. 既存アセットの活用 地域連携RPG「Kitazuna」は、Unityで開発中（進捗約50%）のプロジェクトであり、以下の資産をGenesisに直接流用できる。 ・アイテムDB・インベントリシステム ・ワールド生成・オブジェクト配置ロジック ・ユーザー認証・マルチプレイヤー基盤 ・UIコンポーネント群 これにより、ゼロからの開発と比較して約3ヶ月分の開発工数を短縮できる。 チームとしての実績：AI NPC導入 MinecraftおよびKitazuna（Unity）上にAI NPCを導入し、ユーザーとの自然言語会話を成立させることに成功。GenesisのAIバディ機能の技術的前身であり、「ゼロからの構想」ではなく「既に動いている技術の発展」である。 4-5. 起業家ネットワーク 高崎はピッチ大会への出場等を通じて起業家ネットワークを構築しており、事業化に向けたメンタリングや協業先の開拓が可能な状態にある。
