## Step 5: 修正提案

### A（重要）：採点に大きく影響する修正

#### A-1. 先行研究への言及を追加する

**現状の問題：**
「Stealth Assessment」は2011年にValerie Shuteらが提唱した学術的概念であり、ゲーム内の行動データから学習者の能力を推定する研究は世界的に蓄積がある。提案書はこれを知らないかのように記述しており、竹内PMから「知的誠実さに欠ける」と判断されるリスクがある。

**修正案：** セクション②に以下を追加
```
本アプローチは、Shute (2011) が提唱した "Stealth Assessment"
の概念を発展させたものである。既存研究がデジタルゲーム内の
限定的なタスクでの能力推定に留まるのに対し、本システムは
(1) メタバース内の自由度の高い行動全体からの推定、
(2) 臨床心理士による500人以上の検査データとの対応づけ、
(3) WAIS/WISCの4指標すべてのリアルタイム推定
という3点で先行研究を超える。
```

**影響：** A-1（技術的新規性）で竹内PM +1.0点、他のPMでも+0.5点。全体で約+2〜3点。

---

#### A-2. アセスメント精度の検証方法を明確にする

**現状の問題：**
「if(塾)で実際のWISC検査結果との照合」と記載されているが、竹内PM・村上PMから「どの程度の精度なら成功とするのか基準がない」「n=30で統計的に十分か」と指摘されている。

**修正案：** セクション③の技術的リスク対策に追加
```
精度検証方法：
・if(塾)の生徒30名のうち、既にWISC検査を実施済みの全員を対象に、
  ゲーム行動ログからの推定値と実際の検査値のピアソン相関係数を算出。
・目標精度：各指標においてr ≧ 0.6（中程度以上の相関）を達成。
・Leave-One-Out交差検証により、過学習を防止。
・n=30の制約を認識した上で、効果量（Cohen's d）で有意性を評価。
・MVP段階では全4指標ではなく、PSI（処理速度）とWMI（ワーキングメモリ）の
  2指標に絞り、確実な精度達成を優先する。
```

**影響：** C-2（技術的実現可能性）で竹内PM +1.0点、村上PM +0.5点。全体で約+1〜2点。

山﨑の実績：
・○○（具体的なMinecraftサーバー名/プラグイン名）を開発・運営。
  利用者数○名。

井上の実績：

```

**注意：** これは事実情報が必要です。山﨑・井上の具体的な成果物（GitHubリポジトリURL、サーバー名、利用者数、受賞歴等）を確認してください。

### B（改善）：あると加点される修正

#### B-1. if(塾)での具体的エピソードを1つ入れる

**現状の問題：**
ニーズの検証（B-2）が3/5に留まっている主因は、30名という数字のみで「実際にこの仕組みが効果的である」具体的なエビデンスがないため。

**修正案：** セクション①またはセクション②に追加
```
例えば、if(塾)のある生徒はWISCの処理速度指標（PSI）が極端に低い一方、
知覚推理（PRI）が突出して高かった。既存の教育環境では「作業が遅い子」
として評価されていたが、Minecraftでの建築活動では複雑な空間設計を

#### B-3. IRB申請を「必須」に変更する

**現状の問題：**

研究計画を申請し、承認を得た上でデータ収集を開始する。

**影響：** C-2で村上PM +0.5点。B-5でも+0.3点。全体で約+0.5〜1点。
